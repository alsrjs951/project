# On-Device AI 민원 분석 및 처리 시스템 기획 초안

## 1. 기본 정보
- 프로젝트명: On-Device AI 민원 분석 및 처리 시스템
- 개요: 공공기관 폐쇄망 환경에서 민원 데이터를 안전하게 분석하고, 표준화된 답변 초안을 자동 생성하는 온프레미스 AI 시스템
- 동기: 민감한 민원 데이터의 보안 요구사항을 충족하면서도 민원 처리 속도와 품질을 동시에 높이기 위함
- 예상 결과물: 폐쇄망 내 설치 가능한 민원 분석/답변 생성 웹 서비스(모델, 서빙, UI 포함)

## 2. 개발 배경 및 필요성
- 공공기관 민원 데이터는 민감한 개인정보를 포함하고 있어 외부 클라우드 전송 시 보안 위반 리스크가 큼. 따라서 폐쇄적인 로컬 환경에서의 처리가 필수적임.
- 기존의 단순 키워드 분류 방식으로는 복잡한 민원 맥락을 파악하기 어려움.
- 담당 공무원이 수많은 민원 사례를 일일이 확인하는 데 막대한 시간이 소요됨. 단순 키워드 매칭이 아닌 의미 기반 검색을 통해 과거 유사 사례를 즉각적으로 참조할 수 있는 시스템이 요구됨.
- 대규모 모델 대비 유지비가 저렴하고 특정 도메인에 특화된 sLLM을 활용하여 경제적인 AI 서비스 구현이 가능함.

## 3. 개발 요구사항

### 3.1 데이터 확보 및 학습
- 각 지자체별 `전자민원창구` 게시판에서 공개 민원 및 답변 데이터를 수집하는 크롤러 개발
- 수집된 데이터를 기반으로 오픈소스 sLLM(Gemma 2, Llama 3 등)에 대한 지도 미세조정(SFT, Supervised Fine-tuning) 수행

### 3.2 표준 답변 생성 엔진
- 민원 답변 표준 서식을 시스템 프롬프트로 설계하여 일관된 톤앤매너 유지
- 민원 요지 입력 시, 학습된 데이터를 바탕으로 즉시 고품질의 답변 초안을 생성하는 기능 구현

### 3.3 고성능 온프레미스 서빙 최적화
- 폐쇄망 내 설치 및 구동이 가능한 배포 패키징 제공
- 공공기관 환경에 맞는 안정적 추론 성능 및 운영 편의성 확보

### 3.4 사용자 친화적 UI
- Streamlit 기반의 직관적인 웹 인터페이스 구축
- 필수 화면: 민원 입력창, 답변 생성 결과창, 수정 기능, 복사 기능

## 4. 기술 스택 (초안)
- Model: Gemma 2, Llama 3 등 오픈소스 sLLM
- Fine-tuning: SFT 중심의 도메인 특화 학습
- Frontend/UI: Streamlit
- Deployment: On-Premise + Docker 기반 컨테이너 배포

## 5. 비고
- 공공기관 게시판의 `공개` 데이터를 활용하되, 학습 과정에서 비식별화 처리를 강화하여 개인정보 보호 가이드라인을 준수함.
- 향후 타 지자체나 타 공공기관의 폐쇄망 환경에도 즉시 이식 가능한 Docker 기반 컨테이너화를 고려함.
